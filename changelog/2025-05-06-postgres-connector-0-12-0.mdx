[Conduit Postgres Connector v0.12.0](https://github.com/ConduitIO/conduit-connector-postgres/releases/tag/v0.12.0) is now available, and it will be part of the next Conduit release as a [builtin connector](/docs/core-concepts#built-in-connector), making use of the the new Connector SDK `ReadN` method, and adding validation for the logical replication slot name.

<!--truncate-->

### More information

#### [`ReadN` addition](https://github.com/ConduitIO/conduit-connector-postgres/pull/276)

Since the introduction of [end-to-end batching on the Connector SDK](https://github.com/ConduitIO/conduit-connector-sdk/pull/193), (part of the [Connector SDK 0.12.0](https://github.com/ConduitIO/conduit-connector-sdk/releases/tag/v0.12.0)), source connectors could add support for batching by implementing the `ReadN` method. 

#### [Validating logical replication slot name](https://github.com/ConduitIO/conduit-connector-postgres/pull/280)

This validation was introduced by one of our newest Conduit users. Thanks [Rauan Mayemir](https://github.com/rauanmayemir)!

Rauan reported that publications are fine with special characters in the name, but not replication slots, so with Conduit Connector Postgres [0.12.0](https://github.com/ConduitIO/conduit-connector-postgres/releases/tag/v0.12.0), you can ensure that the replication slot name is valid before starting the connector.


:::tip
For more information, check out the [Postgres Connector documentation](/docs/using/connectors/list/postgres).
:::
