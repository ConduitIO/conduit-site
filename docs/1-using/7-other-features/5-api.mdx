Conduit exposes a gRPC API and a REST API.

## gRPC API

The gRPC API is by default running on port 8084. You can define a custom address
using the CLI flag `-grpc.address`. To learn more about the gRPC API please have
a look at
the [protobuf file](https://github.com/ConduitIO/conduit/blob/main/proto/api/v1/api.proto).

:::info
Conduit uses gRPC streams to communicate with [standalone connectors](/docs/core-concepts#standalone-connector).

Make sure to keep the [api configuration](/docs/configuration) enabled (`--api.enabled`) if you want to use [standalone connectors](/docs/core-concepts#standalone-connector). 

Check out the [Connector Behavior documentation](/docs/developing/connectors/behavior) for more information.
:::


## REST API

The REST API is by default running on port 8080. You can define a custom address
using the CLI flag `-http.address`. It is generated
using [gRPC gateway](https://github.com/grpc-ecosystem/grpc-gateway) and is thus
providing the same functionality as the gRPC API. To learn more about the REST
API please have a look at the [API documentation](https://www.conduit.io/api),
[OpenAPI definition](https://github.com/ConduitIO/conduit/blob/main/pkg/web/openapi/swagger-ui/api/v1/api.swagger.json)
or run Conduit and navigate to
[`http://localhost:8080/openapi`](http://localhost:8080/openapi) to open
a [Swagger UI](https://github.com/swagger-api/swagger-ui) which makes it easy to
try it out.
